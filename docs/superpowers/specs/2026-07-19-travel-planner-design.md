# 私人出行助理(travel-planner)- 设计文档

日期:2026-07-19

## 目标
一个"私人出行助理"网页 App 原型:管理多个出行行程,围绕每次出行提供天气、待办时间线、智能打包清单、交通/住宿、日程、预算、签证提醒等信息。第一阶段为 **UI 原型 + mock 数据**,界面定稿后第二阶段接入真实数据源。

## 关键背景
- 护照国籍由用户在"我的"页自行设置;大部分常见目的地免签,仅当目的地需要签证/电子签时才显眼提醒
- 目标设备:**iPhone 16 竖屏(393×852)为基准**,未来可能扩展 iPhone 折叠屏(展开态双栏)
- 视觉风格:**清新蓝白 iOS 风**(白底 + 天空蓝 #4A90D9 渐变强调色,圆角卡片)

## 阶段划分
- **本期(阶段一)**:全部界面 + mock 数据 + localStorage 持久化,浏览器直接打开可用
- **阶段二(不在本期范围)**:接入真实天气 API、汇率 API、签证数据源、航班动态

## 导航结构(底部 Tab Bar,3 个)
1. **✈️ 行程**(默认首页):行程列表
   - 顶部:标题"我的行程" + 新建按钮(+)
   - "即将出行"大卡片:目的地+国旗 emoji、起止日期、倒计时天数、天气徽章、清单进度徽章
   - "历史行程"灰色小条列表
   - 新建行程:弹层表单(目的地 + 起止日期),提交后加入列表并存 localStorage
2. **📅 日历**:月视图
   - 可左右切换月份;行程日期段用蓝色高亮
   - 日历下方列出当月行程卡片,点击进详情
3. **👤 我的**:App 级工具
   - **护照档案**:国籍(用户自行设置)、护照有效期及到期提醒(距到期 < 9 个月时提示,mock 判断)
   - **汇率换算器**:独立小工具,输入金额双向换算(mock 汇率表)
   - 设置项占位(仅 UI,无功能)

## 行程详情页
板块多,采用**折叠卡片**:每个板块默认一行摘要,点击展开详情;头图与签证状态条常显。

1. **头图**:蓝色渐变,目的地+国旗、起止日期、倒计时
2. **签证状态条**(紧贴头图下方):
   - 免签 → 一行低调小字"🛂 护照免签 90 天"
   - 需签证/电子签 → 黄色警示横幅"⚠️ 该目的地需提前办理签证",且自动在待办时间线插入"提前 X 天办签证"事项
3. **🌤 天气预报**:未来 5 天(图标/高低温/描述),mock 数据
4. **✅ 待办时间线**:按出发日倒排的待办(如 提前30天订酒店 / 提前14天办签证(如需)/ 提前3天换汇 / 前1天打包),每项可勾选,过期未完成的标红
5. **🧳 智能打包清单**:根据目的地天气(热/冷/雨)与行程天数自动生成建议项(mock 规则),可勾选/增删,显示进度 n/m
6. **🚄 交通**:机票/高铁/船票购买入口(携程、飞猪、Trip.com 跳转按钮,原型为示意链接);出行日显示航班动态卡(航班号/值机口/状态,mock)
7. **🏨 住宿**:酒店名、地址、入住/退房时间、确认号(mock,可编辑)
8. **🗓 每日日程**:Day 1–N 时间线,每天若干条安排(时间+活动)
9. **📍 想去清单**:收藏的景点/餐厅列表,可标记"已排入日程"
10. **💰 预算**:分类金额(交通/住宿/餐饮/购物等),**双币显示**(当地币种 + AUD,用 mock 汇率换算),显示总计
11. **🆘 紧急信息卡**:当地紧急电话、本国领事紧急热线(用户自行填写)、保险单号、酒店地址,设计为离线可查(纯本地数据)
12. **🔌 目的地小贴士**:时差、插座制式、小费习惯、当地交通卡(如八达通)

## 数据与状态
- 全部内容来自 `js/data.js` 的 mock 数据,预置:
  - 3 个行程:香港(未来,8月)、东京(未来,10月)、曼谷(历史,3月)
  - 签证规则表:(用户护照国籍, 目的地) → { 类型: 免签/电子签/需签证, 停留天数, 建议提前办理天数 }
  - 汇率表:AUD ↔ HKD/JPY/THB/CNY(固定 mock 值)
  - 每个行程的天气、待办、打包清单、交通、住宿、日程、想去清单、预算、紧急信息、小贴士
- **localStorage** 持久化:勾选状态(待办/打包清单)、新建的行程、编辑过的数据;刷新不丢失
- 阶段二切换真实数据时只替换 `data.js` 的数据来源,视图层不动

## 文件结构(无构建工具,浏览器直接打开)
```
travel-planner/
  index.html        页面骨架 + 各屏容器 + tab bar
  css/base.css      设计变量、重置、通用组件(卡片/徽章/按钮/折叠区)
  css/screens.css   各屏幕布局样式
  js/data.js        全部 mock 数据(行程/天气/签证规则/汇率/贴士)
  js/views-main.js  行程列表页、日历页、"我的"页渲染
  js/views-trip.js  行程详情页各板块渲染
  js/app.js         导航切换、事件绑定、localStorage 存取、入口
```

## 尺寸与适配
- iPhone 16(393×852)竖屏为设计基准
- 桌面浏览器打开时页面居中显示手机比例容器(max-width 430px),带浅色外部背景,方便预览
- 真机全屏自适应
- CSS 预留 `@media (min-width: 600px)` 断点注释位,将来折叠屏展开态实现"列表+详情"双栏;**本期不实现**

## 视觉规范
- 背景 `#F5F8FC`,卡片白色 `#FFFFFF`,主色 `#4A90D9`,渐变 `#4A90D9 → #67B8F7`
- 主文字 `#1B2A41`,次要文字 `#7A8CA5`
- 圆角:卡片 16px,徽章全圆角;阴影浅蓝色调 `rgba(30,80,160,0.08)`
- 系统字体栈(-apple-system / PingFang SC),emoji 做图标,无外部资源请求
- 签证警示横幅:黄底 `#FFF7E0` + 深黄字 `#8A6D1A`

## 验证方式
浏览器 DevTools 切 iPhone 16 视口(393×852)走查:
- Tab 三屏切换、日历翻月、详情页折叠卡展开收起
- 待办/打包清单勾选后刷新仍保留
- 新建行程出现在列表与日历
- 香港行程(免签)只显示低调签证小字;构造一个"需签证"目的地验证黄色横幅与待办自动插入
- 汇率换算器双向换算、预算双币显示正确(按 mock 汇率手算核对)
